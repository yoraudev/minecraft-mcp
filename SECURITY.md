# Security Policy

## Supported Versions

Security updates are provided for the latest version on the default branch.

## Reporting a Vulnerability

1. Do not open a public issue for security vulnerabilities.
2. Use GitHub Private Vulnerability Reporting if enabled.
3. If private reporting is unavailable, contact maintainers through a private channel.
4. Include clear reproduction steps, impact, and affected versions.

## Response Targets

1. Initial acknowledgment target: within 72 hours.
2. Triage target: within 7 days.
3. Resolution timeline depends on severity and complexity.

## Disclosure

Please follow responsible disclosure and allow maintainers time to prepare and release a fix before public disclosure.
